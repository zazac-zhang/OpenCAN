# OpenCAN GUI UI/UX 设计规格

**日期：** 2026-06-06  
**目标：** 打造世界最好用的 CAN/CANOpen 客户端软件  
**目标用户：** 现场调试工程师  
**设计风格：** 工业专业风格（Wireshark/CANalyzer 风格）

---

## 1. 整体布局

### 1.1 四区域布局

```
┌─────────────────────────────────────────────────────────────┐
│                        工具栏 (48px)                         │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│  节点面板  │              主工作区                             │
│  (200px) │           (Tab 切换)                              │
│          │                                                  │
│          ├──────────────────────────────────────────────────┤
│          │              详情面板 (可折叠, 200px)              │
├──────────┴──────────────────────────────────────────────────┤
│                     状态栏 + 日志过滤 (32px)                  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 各区域职责

| 区域 | 职责 | 交互 |
|------|------|------|
| 工具栏 | 连接控制、NMT 快捷操作、全局操作 | 一键式按钮 |
| 节点面板 | 节点树、连接状态、快速导航 | 单击选中、右键菜单 |
| 主工作区 | Tab 切换：CAN 总线 / CANOpen 协议 | 二级 Tab 切换 |
| 详情面板 | 选中节点的 OD 缓存、DS402 状态、SDO 结果 | 跟随节点选择自动更新 |
| 状态栏 | 连接状态、最后操作、错误信息、日志过滤 | 实时更新 |

## 2. Tab 结构

### 2.1 一级 Tab（协议层次）

| 一级 Tab | 定位 |
|---------|------|
| **CAN 总线** | 物理层/数据链路层：原始帧、总线统计、错误帧 |
| **CANOpen 协议** | 应用层：NMT、SDO、PDO、SYNC、EMCY、Heartbeat、DS402 |

### 2.2 CAN 总线（二级 Tab）

| Tab | 内容 |
|-----|------|
| 帧监控 | 所有 CAN/CAN FD 帧列表，类似 Wireshark |
| 总线统计 | 总线负载、错误计数、帧率 |
| 错误帧 | Bus Off、Error Passive、错误详情 |

### 2.3 CANOpen 协议（二级 Tab）

| Tab | 内容 |
|-----|------|
| 网络管理 | NMT 状态、节点管理、启动/停止/重置 |
| SDO 客户端 | SDO 读写、传输历史、结果缓存 |
| PDO 监控 | TPDO/RPDO 数据表、映射解析 |
| DS402 控制 | 状态机、操作模式、运动控制、实时数据 |
| EMCY 日志 | 紧急错误记录、错误代码解析 |
| 心跳监控 | Heartbeat 状态、超时检测 |
| 同步管理 | SYNC 生产者/消费者配置 |

## 3. 功能模块设计

### 3.1 CAN 总线 — 帧监控

**功能：**
- 实时显示所有 CAN/CAN FD 帧
- 表格列：时间戳 | CAN ID | DLC | 数据 (hex) | 周期 | 解码
- 支持暂停/继续、清空、导出
- 点击某帧可在详情面板显示解码内容

**交互：**
- 双击帧 → 高亮同类 CAN ID
- 右键 → 添加到过滤器、复制数据

### 3.2 CAN 总线 — 总线统计

**功能：**
- 总线负载（百分比 + 图表）
- 帧率（帧/秒）
- 错误计数（发送/接收错误）
- 波特率显示
- 总线状态（Active / Warning / Passive / Bus Off）

### 3.3 CAN 总线 — 错误帧

**功能：**
- 错误帧列表：时间戳、错误类型、错误标志
- 错误类型分类：Bit Error、Stuff Error、CRC Error、Form Error、ACK Error
- 错误计数器状态（TEC / REC）

### 3.4 CANOpen — 网络管理

**功能：**
- 节点列表：Node ID | NMT 状态 | 设备类型 | 心跳 | DS402 状态
- 批量操作：全部启动、全部停止、全部重置
- 单节点操作：启动、停止、重置、进入预操作
- 节点扫描（自动发现在线节点）
- 右键菜单：查看详情、发送 NMT、清除节点

### 3.5 CANOpen — SDO 客户端

**功能：**
- SDO 读写面板：
  - 节点 ID（下拉选择或手动输入）
  - Index / Subindex（支持 0x 前缀）
  - 数据类型选择（UNS8/16/32, INT8/16/32, VisibleString, Domain 等）
  - 值输入（根据数据类型自动格式化）
  - 读取 / 写入按钮
- SDO 传输历史：
  - 时间戳 | 节点 | Index:Sub | 类型 | 值 | 结果（成功/失败 + 错误码）
- 收藏夹：
  - 常用 OD 对象快速访问（可自定义）

### 3.6 CANOpen — PDO 监控

**功能：**
- PDO 数据表格：
  - 时间戳 | COB-ID | PDO 类型 (TPDO1-4/RPDO1-4) | 节点 | 数据 (hex) | 解码值
- PDO 映射配置查看：
  - 显示每个 PDO 的映射对象列表
  - 支持在线读取 PDO 映射（通过 SDO 读 0x1A00/0x1600 等）
- PDO 数据解码：
  - 根据映射配置自动解码 PDO 数据字段

### 3.7 CANOpen — DS402 控制

**功能：**
- 状态机面板：
  - 当前状态（FSM 状态文字 + 状态图示意）
  - Status Word 位级显示（每 bit 用标记指示）
  - Control Word 快捷操作按钮（Shutdown / Switch On / Enable / Disable / Fault Reset）
- 操作模式：
  - 模式选择：PP, IP, CSP, CSV, CST 等
  - 模式专用参数配置
- 实时数据：
  - 位置（实际值 / 目标值）
  - 速度（实际值 / 目标值）
  - 扭矩（实际值 / 目标值）
  - 支持周期性刷新（可配置间隔）

### 3.8 CANOpen — EMCY 日志

**功能：**
- 紧急错误列表：
  - 时间戳 | 节点 | 错误代码 | 错误寄存器 | 数据
- 错误代码解析：
  - 自动解析标准 CANOpen 错误代码（0x1000-0xFFFF）
  - 显示错误描述

### 3.9 CANOpen — 心跳监控

**功能：**
- 心跳状态表格：
  - 节点 | 生产者周期 | 消费者配置 | 最后心跳时间 | 状态（在线/超时/丢失）
- 心跳超时报警：
  - 超时节点高亮显示
  - 状态栏提示

### 3.10 CANOpen — 同步管理

**功能：**
- SYNC 生产者配置：
  - 周期设置、启动/停止
- SYNC 消费者统计：
  - 接收计数、频率测量

---

## 4. 详情面板

**功能：**
- 跟随当前选中节点自动更新
- 显示内容：
  - 节点基本信息（ID、NMT 状态、设备类型、厂商 ID）
  - OD 缓存（已读取的对象字典条目）
  - 最近 SDO 操作结果
  - DS402 快速状态（如适用）

---

## 5. 工具栏

**功能：**
- 连接/断开按钮
- 节点扫描按钮
- NMT 快捷操作（全部启动/停止）
- 当前连接状态指示
- 设置入口（波特率、节点 ID 等）

---

## 6. 状态栏

**功能：**
- 连接状态（已连接/未连接 + 后端类型）
- 最后操作结果
- 错误信息（如有）
- 日志过滤器（CAN 日志 Tab 内）

---

## 7. 实现约束

- **GUI 框架：** iced 0.13 (tokio)
- **主题：** 使用 iced 默认主题（Dark）
- **后端通信：** mpsc channel，50ms 轮询
- **架构模式：** Elm 架构（Message → update → view）
